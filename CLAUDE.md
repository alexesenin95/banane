# CLAUDE.md — контекст проекта для Claude Code

## Что это
2D-платформер «Банан против орков» на **Phaser 3** (CDN в index.html). Герой-обезьяна против орков и босса-паука.
Перенесён из чат-прототипа Claude для нормальной разработки.

## Запуск
Нужен локальный веб-сервер (ассеты грузятся из файлов): `npx serve`, затем открыть localhost-адрес.
`play-offline.html` — автономная сборка для быстрой проверки.

## Структура
- index.html — разметка (оверлеи: меню/интро/кат-сцена/инвентарь/конец) + Phaser + js/game.js.
- css/style.css — UI; фон меню через --menuBg.
- js/game.js — ВСЯ логика одним файлом. Секции: история/интро, инвентарь, кат-сцены, genLevel, create/update, spawnEnemy + ИИ, бой (doAttack/damageEnemy/hitEnemy), помощники.
- assets/sprites — герой, враги, снаряды, барьеры, плитка, иконки оружия. assets/bg — параллакс ×3. assets/portraits — Чичо/Доня/орк/герой. assets/ui — фон меню.
- TEX в game.js — манифест ключ→путь, грузится в preload.

## Системы (работают)
Уровни: 20, сгруппированы по биомам (BIOMES + GROUPS): джунгли 1-3, топи 4-7, пещеры 8-12, ледник 13-16, небесные руины 17-20. У биома свои фон/плитка/препятствия/тинт/босс; cave/ice/sky пока через `ref` переиспользуют арт топей/джунглей (свой арт подменяется добавлением ключей в TEX). Боссы на 3/7/12/16/20. Выбор героя: моно ↔ орангутан (HEROES/HK, экран charselect). Процедурная генерация (~20000px, пропасти/платформы/башни). Враги (армия орков): креветка, орк, берсерк, ножеметатель, лучник (eprojs), щитоносец (inv), тролли t_club/t_bone (тяжёлый ближний), троль-камнемёт t_boulder (катящийся валун — группа rocks, fireRock), боссы (spider/boss_troll). Пулы врагов по биому: POOLS + pickType(pool,R). Бой: стомп + оружие. Инвентарь (I). Прокачка на 400 очков. Броня (B): −500 → 5 сек. Кат-сцены (cuts[].spawn). Кино-интро этапа: CINEMATICS (панели c4_* + Ken Burns), играет перед уровнем через startLevelFlow. Барьеры: шипы/колючки (урон), камень (ломается), частокол (твёрдый), ворота (декор) — текстуры из biomeCfg.obst, авто-постановка на землю. Чекпоинт cpX. ADMIN: клавиши [ ] (перед релизом ADMIN=false).
